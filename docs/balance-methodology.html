Balance methodology

Within Parameters — April 2026

How 45 million simulated game runs, three structural fixes, and bounded agent exploration turned a trivially winnable roguelike into a 6/6 balanced system.

Within Parameters is a roguelike visual novel with 64 possible trait combinations (8 positive × 8 negative). Each combination changes the player's starting conditions, per-event costs, and reward effectiveness. The balance question: does every combination produce a meaningfully different experience without making any combination trivially easy or impossibly hard?

Rather than playtest our way to an answer, we built a Monte Carlo simulator, ran parameter sweeps, and let the data tell us where the balance problems were. The entire process took under two hours of compute time and produced a configuration that passes all six validation criteria.

The validation criteria

Six criteria define "balanced." They were designed before any simulation ran, so the goalposts never moved.

The process

Pre-validation
Gemini Deep Research review
Before writing any simulation code, the 64-cell trait matrix was submitted to Gemini Deep Research for mathematical analysis. GDR identified five critical issues: N7 (Exhausted) was mathematically unwinnable, P4 (Steady Hand) trivializing, a rapport feedback loop allowing infinite clock reduction, a null pairing between P1 and N2, and surplus stat farming breaking the score ceiling. All five were fixed in the trait system revision before simulation began.
First run — 640,000 simulations
Trivially winnable
The heuristic agent won 99.7% of all runs regardless of trait combination. Knowledge accumulated to 15–18 per run against a threshold of 8. Clock pressure was cosmetic: expected 7.5 total accumulation against a max of 10. The agent never needed to make a hard choice.
Sweep v1 — 21 million simulations
33 configs, best 4/6
Swept three axes: knowledge threshold (6 values), knowledge reward (3 values), clock base tick (2 values). Zero configs passed all 6 criteria. Three structural failures identified: scoring compression made S-tier unreachable, P6 (Clear-Headed) created trivial combos, and N7 (Exhausted) didn't scale with clock speed.
Sweep v2 — 24 million simulations
3 structural fixes + agent exploration → 6/6
Applied three targeted fixes, re-swept the same grid, then gave the agent bounded freedom to explore. Five exploratory configs later, a configuration passing all six criteria was found.

The first run

The simulator uses a heuristic agent that models competent-but-not-optimal play. It follows priority rules: survive first (if clock is critical, take clock reduction), stay solvent second (if modules are low, take consumables), chase the win condition third (if knowledge is below threshold, take knowledge), and default to knowledge accumulation.

Sweep v1: parametric search

Rather than guess at fixes, we enumerated the parameter space. 33 configurations across three independent axes, each running the full 640,000-iteration Monte Carlo. Total: 21.1 million simulated game runs in 21 minutes.

No combination of the three swept parameters could fix these issues. They were structural problems in the trait definitions and scoring formula, not tunable parameters.

The structural fixes

Three independent fixes, each addressing one failure mode without interacting with the others.

Fix 1
Third-tier surplus scoring
Added a third diminishing-returns tier instead of a hard cutoff at 2 surplus points. This opened the scoring ceiling enough for S-tier to become achievable.
S-tier: 0% → 19.1%
Fix 2
P6: -2 → -1
Halved Clear-Headed's threshold reduction. Still meaningful (threshold 10 instead of 11) without creating trivial combos at low thresholds.
Max combo: 95%+ → 87.1%
Fix 3
N7: additive +0.25
Changed Exhausted from "jitter always fires" to "jitter chance increases by 0.25." Scales naturally with any base tick value instead of being binary.
N7 clock fail: 84% → 4.9%

Sweep v2: fixes + exploration

With fixes applied, we re-swept the same 33-config grid. The best structured config hit 5/6, failing V3 (mean correction at 28.2%, just below the 30% floor). The spec gave the agent freedom to run up to 10 exploratory configs. It used five.

The balance journey — correction rate across phases

The winning exploration adjusted two parameters on the best structured candidate: starting modules from 5 to 6 (one extra cushion) and jitter chance from 0.50 to 0.35 (slightly softer clock). These gave enough breathing room to cross the 30% mean correction floor.

Trait differentiation

The winning config creates deliberate asymmetry. Most positive traits cluster at 23–30% correction rate, creating a baseline of "this game is hard." Two outliers define the extremes of the roguelike experience.

Average correction rate by positive trait
P2 (Quick Study) dominates at 83%. With knowledge rewards set to zero, P2's +1 bonus makes it the only trait that gets knowledge from rewards. Rolling P2 is the lucky roll. This is intentional roguelike design: some rolls are easier, and the scoring cascade rewards players who accept hard rolls.
N4 (Distracted) is the hardest negative at 1–5% correction without P2. Without document knowledge and without reward knowledge, the only source is event choices. P2+N4 compensates at 66%. This is the challenge run.

Most non-P2 runs end in destruction (68–75%): the player reaches the facility but lacks the knowledge to fix the archive AI. Clock failures are rare (1–3%), meaning clock pressure creates tension without being the primary kill condition. This is the right feel: you get there, but you can't solve it.

What we learned

Validate before building content. The simulator proved balance was broken using placeholder data. Fixing config values is cheap; rewriting dialogue to accommodate new stat curves is expensive. Every hour spent on simulation saved days of potential rework.

Structural problems resist parametric solutions. Sweep v1 proved that no combination of three tunable parameters could fix issues in the trait definitions and scoring formula. Recognizing when to stop tuning and start restructuring is the key insight.

Bounded agent exploration works. Giving the coding agent freedom within constraints (10 configs max, specific parameter ranges, clear stop conditions) produced a better result than either full prescription or full autonomy. The agent used 5 of 10 permitted runs to find the winning config.

The Monte Carlo pattern transfers. This approach originated in a sibling project (Holdfast). The simulator architecture, sweep methodology, and heuristic agent design all carried over directly. Spec-driven development creates reusable patterns because the specs themselves are the knowledge transfer mechanism.
